## Contributing to Documentation

When creating new documentation:

1. Add the file to this directory
2. Use clear, concise Markdown formatting
3. Include code examples where appropriate
4. Update this index with a link to your new documentation file

## Documentation Standards

- Use H1 (#) for document title
- Use H2 (##) for major sections
- Use H3 (###) for subsections
- Include code examples with proper syntax highlighting
- Link to other relevant documentation files
- Keep content up-to-date with code changes
- Use kebab-case for all documentation file names (e.g., file-name.md)
